# 🐍 开心贪吃蛇 — 未来功能与设计路线图

> 本文档梳理贪吃蛇项目未来可能扩展的功能方向与设计思路，供后续迭代参考。
> 按优先级分为「短期可做」「中期规划」「远期愿景」三个梯度。

---

## 一、🎮 游戏模式扩展

### 1.1 经典模式（已实现）
当前基础玩法：吃食物 → 变长 → 避免撞墙/撞自己。

### 1.2 闯关模式（关卡制）
- 预设多个关卡，每关有固定的食物目标和障碍布局
- 吃完指定数量食物后进入下一关
- 关卡间有过渡动画和难度递增

### 1.3 限时挑战模式
- 60/90/120 秒倒计时，时间结束时结算分数
- 食物带有不同分值（普通 10 分、金色 30 分、限时出现）
- 紧迫感强，适合短时间游玩

### 1.4 禅模式（无死亡）
- 撞墙从对面穿出（穿墙模式）
- 撞到自己只缩短一节
- 纯粹放松体验，适合休闲玩家

### 1.5 迷宫模式
- 地图中有固定障碍物/墙壁
- 障碍物随机生成或使用预设迷宫
- 考验空间规划能力

### 1.6 传送门模式
- 地图上随机出现成对传送门
- 蛇进入一端从另一端穿出
- 增加策略深度和不确定性

## 二、🧪 道具系统（Power-ups）

吃食物时有概率在地图上生成道具，持续一定时间后消失。

| 道具 | 图标 | 效果 | 持续时间 |
|------|------|------|----------|
| 🚀 加速 | ⚡ | 移动速度 ×1.5 | 10 秒 |
| 🐢 减速 | 🧊 | 移动速度 ×0.5，分数 ×2 | 10 秒 |
| 👻 穿墙 | 🌫️ | 撞墙从对面穿出 | 8 秒 |
| 🛡️ 护盾 | 💎 | 撞自己不死，改为缩短 | 8 秒 |
| 🧲 磁铁 | 🧲 | 附近食物自动吸引 | 12 秒 |
| ✂️ 缩短 | ✂️ | 蛇身立即缩短 3 节 | 即时 |
| 💰 双倍分 | ✨ | 吃食物分数 ×2 | 15 秒 |
| 🐉 穿身 | 🔮 | 可穿过自己身体 | 10 秒 |

## 三、🏔️ 关卡与地图系统

### 3.1 地图配置化
- 用 JSON/TOML 文件定义关卡布局
- 包含：网格大小、障碍物坐标、初始位置、食物数量目标
- 支持社区自定义关卡

### 3.2 内置关卡包
- 5~10 个由简到难的预设关卡
- 主题命名（如"新手村"、"荆棘迷宫"、"死亡螺旋"）

### 3.3 关卡编辑器
- 可视化拖拽编辑障碍物
- 导出/导入关卡配置文件
- 预览与测试功能

## 五、🏆 成就系统

### 5.1 游戏成就

| 成就 | 条件 |
|------|------|
| 🍼 初次捕食 | 第一次吃到食物 |
| 🐍 小蛇成长 | 蛇身长度达到 10 |
| 🐲 巨龙之躯 | 蛇身长度达到 50 |
| ⚡ 闪电反应 | 单次游戏内连续 20 个食物不死亡 |
| 🎯 精准操作 | 贴墙移动超过 50 格未碰撞 |
| 🏅 百分达人 | 单局得分超过 1000 |
| 👑 排行榜之王 | 最高分进入本地 Top 3 |
| 🧘 禅意大师 | 禅模式下达到 100 分 |
| ⏱️ 极限挑战 | 限时模式下达到 200 分 |
| 🧩 迷宫征服者 | 通过 5 个迷宫关卡 |

### 5.2 成就系统设计
- 成就数据持久化存储（localStorage / SQLite）
- 成就弹窗提醒 + 音效
- 成就展示面板

## 六、⚙️ 技术架构优化

### 6.1 代码架构重构
当前 `snake.py` 为单文件单类，未来可考虑：

```
snake_game/
├── src/
│   ├── core/
│   │   ├── game_state.py      # 游戏状态机
│   │   ├── snake.py           # 蛇实体
│   │   ├── food.py            # 食物实体
│   │   └── collision.py       # 碰撞检测
│   ├── systems/
│   │   ├── render.py          # 渲染系统
│   │   ├── input.py           # 输入处理
│   │   ├── audio.py           # 音频系统
│   │   └── particle.py        # 粒子系统
│   ├── data/
│   │   ├── levels/            # 关卡配置
│   │   ├── themes/            # 主题配置
│   │   └── storage.py         # 数据持久化
│   └── ui/
│       ├── menu.py            # 主菜单
│       ├── hud.py             # 游戏内 HUD
│       └── screens.py         # 各界面
├── main.py                    # 入口
└── assets/                    # 资源文件
```

### 6.2 状态机模式

```
MAIN_MENU → PLAYING → PAUSED → GAME_OVER → MAIN_MENU
                ↓
            SETTINGS / ACHIEVEMENTS / LEVEL_SELECT
```

每个状态职责清晰，便于扩展新界面。

### 6.3 事件总线
- 游戏内部事件解耦（`FOOD_EATEN`、`GAME_OVER`、`ACHIEVEMENT_UNLOCKED`）
- 各子系统通过事件订阅通信
- 便于添加新功能而无需修改核心逻辑

### 6.4 配置化
- 将所有游戏参数（速度、网格、颜色、分值）集中到配置文件
- 支持运行时热切换主题/难度
- JSON Schema 校验配置文件合法性

### 6.5 数据持久化
- 最高分（已实现）
- 游戏设置（主题、音量、语言）
- 成就解锁状态
- 游戏统计数据（总游玩次数、总时长、总吃食物数）
- 支持云同步（远期）

## 九、🏅 竞技与社交（远期愿景）

### 9.1 双人对战（本地）
- 同一网格，两条蛇互相竞争
- 对方蛇身也构成障碍
- 可以逼对手撞到自己或墙壁
- 分屏或同屏显示

### 9.2 AI 对手
- 简单 AI：随机游走 + 避开墙壁
- 中等 AI：向食物方向移动 + 基本避障
- 困难 AI：路径规划 + 策略围堵

### 9.3 在线排行榜
- 提交分数到服务器
- 全球 / 好友排行榜
- 每日挑战 + 排名

### 9.4 回放分享
- 记录每局操作序列
- 生成回放文件
- 分享给好友观看
