# Confirmed Plan: 多关卡设计与独立地图

## User Goal
将塔防游戏从单地图20波改为5个独立关卡，每个关卡有独立地图。

## Confirmed Requirements
1. **5个独立关卡**，每个关卡有独立的地图（路径配置）
2. 每个关卡有独立的波数配置（每关4波，共20波）
3. 关卡递进系统（完成一关 → 自动进入下一关）
4. UI显示当前关卡信息
5. 通过全部5关 → 胜利画面

## Scope
- 文件：`examples/html5_canvas_tower_defense/index.html`
- 新增 `LEVELS` 配置数组
- 修改游戏主类 `TowerDefenseGame`，新增关卡状态管理
- 修改 UI 以显示关卡信息

## Non-Goals
- 不改防御塔类型和机制
- 不改敌人类型和机制
- 不改美术风格和音效系统
- 不新增塔/怪物类型

## Level Design

| 关卡 | 名称 | 地图风格 | 波数 |
|------|------|---------|------|
| 第1关 | 边境巡逻 | 简单直线型 | 4波 |
| 第2关 | 峡谷通道 | S型弯曲 | 4波 |
| 第3关 | 十字路口 | 十字交叉型 | 4波 |
| 第4关 | 蛇形回廊 | 蛇形折返 | 4波 |
| 第5关 | 最后堡垒 | 螺旋复杂型 | 4波 |

## Implementation Plan
1. 定义 `LEVELS` 数组，包含5个关卡的名称、路径、波数、起始资源配置
2. 添加 `currentLevel` 状态属性
3. 修改 `startNextWave` 逻辑：波次完成后检测是否进入下一关
4. 修改 `restart` 逻辑支持关卡重置
5. 修改 `render` 方法适配不同路径
6. 修改 UI 栏显示当前关卡/总关卡
7. 修改胜利条件：通关全部5关
